# DevTier 기획 문서

---

## 1. 핵심 컨셉

GitHub 잔디(contribution) 데이터로 한국 개발자 전투력 점수를 계산하고 티어를 부여하는 서비스.
백준 온라인 저지가 2026년 4월 28일 종료되면서 개발자들이 실력을 증명할 객관적 지표가 사라진 공백을 GitHub 잔디로 채운다.
GitHub README에 뱃지 삽입 → 바이럴 유입 구조.

---

## 2. 전투력 점수 공식

```
score = 총잔디×1 + 현재스트릭×3 + 최대스트릭×2 + 잔디밀도(%)×100 + 피크강도×0.5
```

---

## 3. 티어 구조

| 티어 | 기준 | 색상 |
|------|------|------|
| 챌린저 | 절대 상위 100명 | `#FF4655` |
| 다이아 1~4 | 상위 1~5% | `#56C8D8` |
| 플래티넘 1~4 | 상위 5~15% | `#5AC9A6` |
| 골드 1~4 | 상위 15~30% | `#FFD700` |
| 실버 1~4 | 상위 30~50% | `#C0C0C0` |
| 브론즈 1~4 | 상위 50~100% | `#CD7F32` |

숫자 낮을수록 높은 등급 (1이 최고)

---

## 4. 기술 스택

- **풀스택:** Next.js 14 App Router (Vercel 배포)
- **DB:** Supabase (PostgreSQL)
- **GitHub 데이터:** GitHub GraphQL API
- **뱃지:** SVG 동적 생성

## 5. 디자인 시스템

### 테마
- 다크모드 전용, GitHub Primer 디자인 시스템 기반
- 배경: `#0d1117` / 서피스: `#161b22` / 보더: `#30363d`
- 텍스트: `#e6edf3` (기본), `#8b949e` (서브)
- border-radius: 6px / font: system-ui + monospace

### 뱃지 스펙

**무료 — 카드형 (180×56)**
- 티어별 3D SVG 아이콘 + idle 애니메이션 + stroke animation
- 티어 색상이 테두리로 표현

**프로 전용 — 스탯 카드형 (280×90)**
- 세부 지표 포함: 잔디 수, 현재 스트릭, 백분위

### 티어별 아이콘 + 애니메이션 (v0.4.0 적용)

| 티어 | 아이콘 | UI 애니메이션 | 뱃지 SVG 애니메이션 |
|------|--------|--------------|-------------------|
| 챌린저 | 왕관 | shimmer sweep + border pulse | v0.5.0 |
| 다이아 | 젬 | rotate-shimmer | v0.5.0 |
| 플래티넘~브론즈 | 메달/크리스탈 | glow-pulse | v0.5.0 |

### 공통 UI 애니메이션 (v0.4.0)

| 위치 | 효과 |
|------|------|
| 페이지 진입 | fade-in-up (stagger) |
| 점수 표시 | count-up (0 → 실제값) |

## 6. 수익화 전략

### 핵심 원칙
> 지금 단계에선 수익보다 확산이 먼저. 뱃지가 GitHub에 최대한 많이 깔려야 서비스 가치가 생긴다.

### 단계별 수익화

**1단계 — 확산 (뱃지 1만 개까지)**
- 전부 무료
- 뱃지에 "DevTier" 로고가 마케팅 자산

**2단계 — 개인 프리미엄**
- 가격: ₩990~1,490/월 (₩2,900은 확산을 막는 가격)
- 유료 기능:
  - 스탯 카드형 뱃지 (세부 지표 포함)
  - 점수 추이 30일 그래프
  - 뱃지 커스텀 색상/스타일
  - 비공개 모드

**3단계 — B2B (유저 충분히 쌓인 후)**
- 채용 연동: 챌린저/다이아 유저 프로필을 기업에 노출, 열람 비용 과금
- 교육사/부트캠프: "수료생 평균 DevTier 골드" 증명 자료 판매
- 개발자는 무료, 기업이 돈 내는 구조

### 하지 말아야 할 것
- 초기 유료 플랜 강조 → 바이럴 죽임
- 인스타 광고 → 개발자 타겟에 ROI 없음
- Product Hunt 너무 이른 런칭

## 7. 시장 진입 전략

### 타이밍
백준이 2026년 4월 28일 종료. 커뮤니티 감정적 공백이 2~3주 안에 식음. 그 안에 "나를 증명할 새로운 방법"으로 포지셔닝 필요.

### 단계별 액션

**지금 당장 — MVP 완성**
- 완성 기준: username 입력 → 티어 결과 → 뱃지 복사 → README에 붙여넣으면 뱃지 표시
- Phase 4 완성

**MVP 완성 직후**
- 아는 개발자 단톡방에 공유 (홍보 톤 X, 피드백 요청 톤으로)
  ```
  "백준 없어졌잖아요, 저 GitHub 잔디로 티어 매기는 거 만들고 있는데
  써보고 피드백 줄 수 있어요?"
  ```
- 목표: 뱃지를 GitHub에 달아주는 사람 5~10명
- velog에 SEO 글 1개: "백준 없어진 후 GitHub로 개발 실력 증명하는 법"

**2~4주**
- 피드백 1~2개 빠르게 반영 (초기 유저 참여감 유도)
- 배치 스크립트 돌려서 한국 개발자 데이터 수집 시작
- 티어 카드 이미지 다운로드 기능 추가 (트위터 인증샷 문화 유도)

**한 달 이후**
- 데이터 쌓이고 뱃지 퍼지기 시작하면 X(트위터) 광고 소액 테스트

### 바이럴 구조
```
뱃지 GitHub에 달림 → README 보는 사람이 "저게 뭐야?" 클릭
→ 결과 페이지 유입 → 본인도 뱃지 달기 → 반복
```
GitHub 자체가 유통 채널. 커뮤니티 없어도 이 구조가 돌아가면 됨.
